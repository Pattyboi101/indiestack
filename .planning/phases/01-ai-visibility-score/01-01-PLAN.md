1. In db.py, add a new async function `get_citation_percentile(db, maker_id, days=30)` after the existing `get_total_agent_citations` function (around line 1886). Use the PERCENT_RANK() window function partitioned by category_id. The function should:
   - Join tools with agent_citations using LEFT JOIN (so tools with 0 citations are included)
   - Filter to approved tools only
   - Use datetime('now', '-{days} days') for the time window (parameterized, same pattern as existing functions)
   - Return the maker's best-performing tool: {'name': str, 'citations': int, 'percentile': int}
   - Handle edge case: if the maker's best tool has 0 citations, return percentile as None (NOT 0) so the dashboard can show "No data yet"
   - Handle edge case: if maker has no tools at all, return {'name': '', 'citations': 0, 'percentile': None}

   Use this SQL structure:
   ```sql
   WITH tool_citations AS (
       SELECT t.id, t.name, t.category_id,
              COUNT(ac.id) as cite_count
       FROM tools t
       LEFT JOIN agent_citations ac ON ac.tool_id = t.id
           AND ac.created_at >= datetime('now', ?)
       WHERE t.status = 'approved'
       GROUP BY t.id
   ),
   ranked AS (
       SELECT id, name, category_id, cite_count,
              PERCENT_RANK() OVER (PARTITION BY category_id ORDER BY cite_count) as pct_rank
       FROM tool_citations
   )
   SELECT r.name, r.cite_count, CAST(r.pct_rank * 100 AS INTEGER) as percentile
   FROM ranked r
   JOIN tools t ON r.id = t.id
   WHERE t.maker_id = ?
   ORDER BY r.cite_count DESC
   LIMIT 1
   ```
   After fetching the row, check: if row and row['cite_count'] > 0, return with percentile. If row but cite_count == 0, return with percentile=None. If no row, return empty dict.
