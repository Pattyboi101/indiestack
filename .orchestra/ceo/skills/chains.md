# Skill Chains

Common multi-step workflows. Execute sequentially, loading each skill JIT.
Mask previous skill's output when entering the next phase.

## Build a New Feature
1. `brainstorming` ‚Üí design approval
2. `writing-plans` ‚Üí implementation steps
3. `test-driven-development` ‚Üí red-green-refactor per step
4. `verification-before-completion` ‚Üí confirm everything passes
5. `requesting-code-review` ‚Üí peer review before merge
6. `finishing-a-development-branch` ‚Üí merge/PR/cleanup

## Fix a Bug
1. `systematic-debugging` ‚Üí root cause analysis
2. `test-driven-development` ‚Üí regression test + fix
3. `verification-before-completion` ‚Üí confirm fix holds

## Ship / Deploy
1. `verification-before-completion` ‚Üí all tests pass, no regressions
2. `finishing-a-development-branch` ‚Üí merge or PR

## Polish a Page
1. `critique` ‚Üí identify what needs improvement
2. `frontend-design` ‚Üí implement improvements
3. `polish` ‚Üí final detail pass

## Major Refactor
1. `brainstorming` ‚Üí scope and approach
2. `using-git-worktrees` ‚Üí isolated workspace
3. `writing-plans` ‚Üí step-by-step plan
4. `test-driven-development` ‚Üí refactor with test safety net
5. `requesting-code-review` ‚Üí review before merge
