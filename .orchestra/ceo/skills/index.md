# Skill Index

## BUILD ‚Äî New features, components, creative work
- `brainstorming` ‚Üí Explore intent, propose approaches, get design approval. MANDATORY before implementation.
- `writing-plans` ‚Üí Create bite-sized implementation steps with exact file paths and failing tests.
- `test-driven-development` ‚Üí Write failing test first, then minimal code to pass. Non-negotiable.
- `subagent-driven-development` ‚Üí Fresh subagent per task with code review between tasks.
- `frontend-design` ‚Üí Production-grade frontend with high design quality.

## FIX ‚Äî Bugs, test failures, unexpected behaviour
- `systematic-debugging` ‚Üí 4-phase root cause analysis: investigate ‚Üí pattern ‚Üí hypothesis ‚Üí fix.
- `test-driven-development` ‚Üí Write regression test for the bug, then fix.
- `dispatching-parallel-agents` ‚Üí Multiple independent failures? Deploy parallel agents.

## SHIP ‚Äî Deployment, review, branch completion
- `verification-before-completion` ‚Üí Run commands, read output, confirm success BEFORE claiming done.
- `requesting-code-review` ‚Üí Dispatch code-reviewer subagent before merge.
- `receiving-code-review` ‚Üí Verify feedback is correct before implementing.
- `finishing-a-development-branch` ‚Üí Structured options: merge, PR, or cleanup.
- `using-git-worktrees` ‚Üí Isolated workspace for feature branches.

## DESIGN ‚Äî Frontend, UI/UX, visual polish
- `frontend-design` ‚Üí Build distinctive interfaces, avoid generic AI aesthetics.
- `critique` ‚Üí Evaluate visual hierarchy, information architecture, emotional resonance.
- `polish` ‚Üí Final quality pass: alignment, spacing, consistency, detail.
- `audit` ‚Üí Comprehensive audit: accessibility, performance, theming, responsive.
- `simplify` ‚Üí Strip to essence, remove unnecessary complexity.
- `bolder` ‚Üí Amplify safe/boring designs for visual impact.
- `quieter` ‚Üí Tone down aggressive/loud designs.
- `colorize` ‚Üí Add strategic colour to monochromatic interfaces.
- `animate` ‚Üí Purposeful micro-interactions and motion.
- `adapt` ‚Üí Cross-device, cross-platform consistency.
- `harden` ‚Üí Error handling, i18n, edge cases. Production-ready.
- `optimize` ‚Üí Loading speed, rendering, bundle size.
- `onboard` ‚Üí First-time user experience, empty states.
- `clarify` ‚Üí Improve unclear UX copy, error messages, labels, instructions.
- `delight` ‚Üí Add personality, joy, memorable touches to functional interfaces.
- `extract` ‚Üí Consolidate reusable components and patterns into design system.
- `normalize` ‚Üí Match design system, ensure consistency across pages.

## OPERATE ‚Äî Monitoring, stats, scheduling, maintenance
- `newsletter` ‚Üí Weekly personal newsletter via Telegram.
- `weekly-stats` ‚Üí IndieStack stats digest via Telegram.
- `loop` ‚Üí Run any prompt/command on a recurring interval.
- `schedule` ‚Üí Cron-scheduled remote agents.
- `save-tokens` ‚Üí Audit project for token-saving opportunities.
- `claude-api` ‚Üí Build with Anthropic SDK / Claude API.
- `update-config` ‚Üí Modify settings.json for persistent automation.
