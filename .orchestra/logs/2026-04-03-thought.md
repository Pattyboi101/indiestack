# Provocation Thought — 2026-04-03

## Prompt: Maker of a documentation tool discovers their IndieStack listing

### What would make them claim immediately?
The current unclaimed teaser says "AI agents are recommending this tool — claim to see how many times."
**This is weak.** A maker doesn't know if it's 5 or 500. The number IS the hook.

Hypothesis: showing "AI agents recommended this tool 127 times this month" on unclaimed pages
would drive 3-5x more claim conversions than hiding the number entirely. The specific number
creates urgency; "claim to see" assumes they care enough to act without knowing if it matters.

### What would make them pay $49/mo?
Seeing which search queries led to their tool being recommended.
"Developers searched: 'self-hosted docs', 'open source wiki', 'documentation tool nodejs'"
That's keyword gold. That's worth $49/mo to any serious maker.

### What would make them share publicly?
A shareable "Recommended by AI agents" badge with their count.
Something they can put in their README: "[![Recommended by AI agents](badge.svg)](indiestack.ai/tool/xxx)"

### What would annoy them?
1. Missing install_command → tool ranks lower in search (the -40 penalty)
2. Wrong category (very common for cross-category tools)
3. Stale GitHub description used as tagline
4. No tags → tool doesn't appear in tag searches

### Actionable idea (under 30 min, revenue path exists):
**Show the count on unclaimed pages.** Gate the breakdown (which queries, success rate)
behind claiming. Gate the query-level data behind Pro.

Current flow: mystery → claim → see count (free) → see queries (Pro)
Better flow: see count → claim for breakdown → queries are Pro

This respects the freemium hook while giving makers enough signal to bother claiming.

**Decision**: Log for Patrick to evaluate — changes the claim UX which is a product decision.
