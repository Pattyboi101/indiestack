# Research Log — 2026-04-03

## Search Quality Fix (Iteration 1)

**Issue found**: `_cat_term` used raw first meaningful query term for category boost,
but category names don't always contain the query term:
- "cron job scheduler nodejs" → `_cat_term = "cron"` → no match against "Background Jobs"
- "oauth for nextjs" → `_cat_term = "oauth"` → no match against "Authentication"

**Fix**: Added `_CAT_SYNONYMS` dict (85 entries) mapping query terms to category-name
fragments. "cron" → "background", "oauth" → "authentication", "checkout" → "payments" etc.
Also added NEED_MAPPINGS entries for "database", "background", "hosting" categories.

**Commit**: 6d0f286

## Search Quality Fix (Iteration 1, Session 2)

**Issue found**: The category synonym scan used `_meaningful[0]` (first term) regardless of
whether it had a synonym. "self hosted auth" → first term "self" → no synonym → category boost
goes to `LIKE '%self%'` → no match. Auth tools lose 100-point boost.

**Fix**: Changed to scan ALL meaningful terms and use the FIRST one with a known `_CAT_SYNONYMS`
entry. Falls back to first term only when no synonym exists anywhere in the query.

Also expanded `_CAT_SYNONYMS` from 85 to 114 entries with direct category fragments:
"auth" → "authentication", "payment" → "payments", "monitor" → "monitoring", "deploy" →
"deployment", "ci" → "ci-cd", "form" → "forms", "cache" → "caching", etc.

Now "nodejs oauth", "self hosted auth", "self hosted monitoring" all get correct category boosts.

## Search Quality Fix (Iteration 3, Session 3)

**Issue found**: Framework qualifier queries like "auth for nextjs", "payments django" were broken.
FTS5 tokenises "Next.js" as "next"+"js" (separate tokens). A query for "nextjs" would never
FTS-match tool descriptions containing "Next.js". Result: zero or irrelevant tools.

**Fix**: Added `_FRAMEWORK_QUERY_TERMS` dict (18 frameworks) and auto-detection logic in
`search_tools`. When a query contains a framework term alongside non-framework terms:
- Strip framework terms from FTS query (so FTS stays tight on the actual category)
- Add `frameworks_tested LIKE '%nextjs%'` filter to narrow to compatible tools
- Only activates when no explicit `frameworks=` param passed

"auth for nextjs" → FTS: `"auth"*`, filter: `frameworks_tested LIKE '%nextjs%'`
"payments django" → FTS: `"payments"*`, filter: `frameworks_tested LIKE '%django%'`

Safe cases: single framework-only query ("nextjs"), stop-word-only query — no splitting.

**Also fixed**: 22 orphaned commits from previous autonomous sessions were rescued from a
detached HEAD state and fast-forwarded onto master. Rebased and pushed.

**Also fixed**: Unclaimed tool pages now show actual AI recommendation count ("127 times")
instead of mystery "claim to see how many times" — the number is the hook for maker claims.

**Commits**: 6b8f43d (unclaimed count), 4bcf748 (framework query fix)
