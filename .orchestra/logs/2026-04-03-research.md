## Search Quality Fix (Iteration 1)

**Issue found**: `_cat_term` used raw first meaningful query term for category boost,
but category names don't always contain the query term:
- "cron job scheduler nodejs" → `_cat_term = "cron"` → no match against "Background Jobs"
- "oauth for nextjs" → `_cat_term = "oauth"` → no match against "Authentication"

**Fix**: Added `_CAT_SYNONYMS` dict (85 entries) mapping query terms to category-name
fragments. "cron" → "background", "oauth" → "authentication", "checkout" → "payments" etc.
Also added NEED_MAPPINGS entries for "database", "background", "hosting" categories.
